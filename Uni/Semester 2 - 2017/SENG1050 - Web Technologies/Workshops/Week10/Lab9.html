Lab 8 Answers

Task 1

Is there progress? - Both examples could be fixed with an absolute or
overall progress bar. The first example could convince the user that their PC has
locked up, while the second is provides no indication of when the download will conclude.

I want them all! - The example uses radio buttons disguised as checkboxes for visual
consistency, but instead causes confusion when a user tries to select mutually-exclusive
options. This could be fixed by greying out options when their mutually-exclusive
counterparts are selected.

Fun with scrolling! - The first example uses nested scrollbars, and can be fixed by
expanding the inner window so that the inner scrollbar is unneccessary. The second combines
horizontal and vertical bars, and can be fixed by listing all of the options vertically, or by
extending the window horizontally so that all information appears on the screen.

More tabs please! - The first example has a ridiculous amount of tabs clogging the interface.
Drop-down menus would be a better option here. Also, remove the unnecessary and distracting graphics.

Helpful tips! - Providing help for the user is important, but it must be solicited.
Excessive and/or lengthy rollover descriptions will make the page frustrating to navigate.

Green good - red bad! - Using colour should never be used in buttons to suggest choice.
Plain neutral colours do not suggest that one option is preferred over another (in the provided example,
wiping all the records is green/affirmative, but this could be a permanent and costly mistake.)
