Task 1 - Creating XML Data

  1. Look at the following Valid XML document bibliography.xml. 
    
<?xml version="1.0" encoding="utf-8"?><!DOCTYPE bibliography [
<!ELEMENT bibliography (book+)>
  <!ELEMENT book (author+, title, pub?, year, URL*)>
  <!ELEMENT author (
	(givenname | nickname)?, surname)>
  <!ELEMENT givenname (#PCDATA)>
  <!ELEMENT nickname (#PCDATA)>
  <!ELEMENT surname (#PCDATA)>
  <!ELEMENT title (#PCDATA)>
  <!ELEMENT pub (#PCDATA)>
  <!ELEMENT year (#PCDATA)>
  <!ELEMENT URL (#PCDATA)>
]>

<bibliography>
   <book>
	<author>
		<nickname>Fred</nickname>
		<surname>Flintstone</surname>
	</author>
	<author>
		<givenname>Helen</givenname>
		<surname>James</surname>
	</author>
	<title>"Statistical Database Disclosure"</title>
	<year>2006</year>
	<URL>http://some.webaddress.com</URL>
	<URL>http://another.webaddress.com</URL>
   </book>
   <book>
	<author>
		<surname>Simpson</surname>
	</author>
	<title>"Homer's Odyssey"</title>
	<pub>Springer Verlag</pub>
	<year>2003</year>
   </book>
</bibliography>
  2. Make sure you understand how the included DTD is working to define the structure of the XML. You may find it useful to draw an XML Tree, as shown in the Lecture notes, to help you visualise the structure.
  3. Now think of how you can modify the DTD to include information for a journal article as well as for a book. The bibliography entry for a journal article contains the following information: Author(s) names (given name, surname, nickname, article title, journal name, volume number, issue number, page numbers (e.g. 10-17), year, URL). Then add to the XML to include at least one journal article.
  4. Validate your XML against the DTD by submitting it to an XML Validator.

Task 2 - Creating XML Data

  1. Create a new plain-text file called airline.xml.
  2. Using your favourite text editor, copy the following information (representing one "booking") into airline.xml.

  3. Add tags to make airline.xmla well-formed XML document.
    • The root tag should be <bookings>.
    • All the booking information for one passenger should be enclosed in a <booking> tag.
    • While adding tags, give some consideration to the structure of the data. For example, a passenger name consists of three parts - family name, given name, and initials - while departure and arrival data consists of an airport code and a time.

Task 3 - Create a Document Type Definition

  1. Add an internal Document Type Definitions (DTD) to airline.xml.
    • There are potentially many (but possible zero) bookings.
    • Addresses and phone numbers are optional.
    • There is at least one leg in each bookings.
    • Class and aircraft information is optional.
  2. Validate your XML against the DTD by submitting it to an XML Validator.

Task 4 - Create DTD and XML file for data transfer

(Perhaps to be completed as homework if there is not sufficient time in laboratory.)

Coles has hired you to transfer all of its customer records from the Oracle database to MySQL database. You decide to use XML for transferring the records. The following information is stored for each customer: Name, Date of Birth, Address, Phone number, Flybuys number and a sequence of purchase records. In each purchase record the stored information is: Purchase date, Purchase time, Store ID, Total amount and the list of items purchased. For each purchased item the following information is saved: Item code, Item name, Unit price, Quantity, Item price. The following rules are imposed:

  • Each customer record has potentially many (but possible zero) purchase records.
  • Address of customer is optional but phone number is mandatory.
  • There should be at least one item in each purchase.

Your first step is to get the following steps done for data transfer:

  1. Create an external DTD for representing the customer purchase history.
  2. Create a valid XML file with at least two customer records where each customer record contains at least two purchase records.
  3. Validate your XML against the DTD by submitting it to an XML validator.
