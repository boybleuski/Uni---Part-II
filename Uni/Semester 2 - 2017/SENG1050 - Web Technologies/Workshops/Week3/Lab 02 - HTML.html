Task 1 - Create an HTML Template

As all HTML pages are required to have a certain minimum structure, it is a good idea to keep a template file containing that structure. You can then copy the file each time you need to create a new Web page.

  1. Start your favourite plain-text editor. This could be notepad, or notepad++, or some other editor designed to produce .txt files. Do not use a "word processor", such as Microsoft Word or a web authoring tool like Frontpage, or Dreamweaver. The special formatting information used by word processors will make your HTML files invalid.
  2. Cut-and-paste the following line into your text editor: 
    <!DOCTYPE html>
    These tell the Web browser that you are using HTML 5.
  3. Below those lines, cut-and-paste the following: 
    
<html>
   <head>
	<meta charset="UTF-8">
   </head>

   <body>
	
   </body>

</html>
    You now have the basic structure of a HTML document.
  4. Two final touches - you should add a title inside the head of the document 
    
<head>
	<title>Put the Title Here</title>
</head>
    and you should add a level-1 heading inside the body 
    
<body>

	<h1>Put the Most Important Heading Here</h1>

</body>
  5. Create a new directory in U-drive. Perhaps you could call it comp1050. Do not create a directory in C-drive, My Documents, or the desktop.
  6. Save the file you have created into your new directory. Call the file template.html.

Open the file in your favourite Web browser (either start the Web browser and use the File menu, or simply double-click on the file). It should look something like this.

Be careful to ensure the file is saved as template.html and not template.html.txt - some text editors will do the latter by default when creating a new file. To avoid this, be sure to set the File Type to all files.

It is also a good idea to slightly change the settings on Microsoft's File Explorer. From the menubar, select Tools then Folder Options. Select View, and make sure the box labelled Hide extensions for known file types is not ticked. This will enable you to see full filenames, including whether the suffix is .html or .html.txt.

Task 2 - Adding HTML Tags to Your Content

  1. Make a copy of template.html and call it index.html. Open this new file in your favourite plain-text editor.
  2. Change the content of the title (<title>) and level-1 heading (<h1>) tags to be My Favourite Things.
  3. Make a list of at least 3 of your favourite things immediately after the level-1 heading.
  4. Format each favourite thing into a level-2 heading (<h2>).
  5. Add paragraph tags (<p>) around each paragraph describing a favourite thing. Use <strong> and <em> tags to strongly emphasize and emphasize some phrase or words in your description.
  6. Under each paragraph, start an unordered list (<ul>). Each list should contain two items (<li>): the URL of the Web page about that favourite thing, and the original URL of the image related to that favourite thing. Each URL should be hyperlinked to its destination: in other words, <a href="url">url</a>.
  7. After the unordered-list add a quotation related to your favourite item using <blockquote>. You should use the 'cite' attribute to add a URL referning to the source of the citation.
  8. After each blockquote, place an inline image tag linking to the local copy of the image you downloaded. As an image tag is an inline tag and must be inside a block tag, place a division tag around the image tag. For example: 
    
<div>
   <img src="bear.gif" alt="Teddy Bear" />
</div>
  9. At the bottom of the page, add a horizontal rule (<hr>), then add an address block (<address>) and in it place your contact details (name, email address, student number and degree program). Separate each logical line of the address with a break (<br>) tag.

Open the file in your favourite Web browser. It should look something like this.

If you have time, experiment with some other tags introduced in Lecture 2. You might include

  • Your date of birth using <sup> tag to format the date
  • A sample of verse (perhaps from the Internet) referring to your favourite things formatted using <blockquote>
  • A definition list
  • Abbreviations
  • Special Characters: <, > &
  • A sample of computer code (say Java or C++) with input and output
  • Any other use of tags from Lecture 2, such as input instructions, definitions or citations

Task 3 - Learn to link your HTML documents

Linking to Identifiers

  1. Create another copy of index.html and call it index2.html. Open this new file in your favourite plain-text editor.
  2. At the top of the page, (under the <h1> heading) create a <div> element.
  3. Inside the <div> element create anchors (<a>) - one for each of your favourite things (<h2>). Insert unique names preceded by # sign in the href attribute field of each anchor element.
  4. Now complete the linking by putting the chosen names in the corresponding <h2> elements' id attribute field.
  5. Verify that your links are working correctly.
  6. Finally, add one <hr> before and after the <div> element you created for links.

Linking among different files

  1. Create one separate html file for each of your favourite things copying from the original index.html. Each file will contain information about one of your favourite things starting from the <h2> heading to the image of your that particular favourite thing. Rename each of the files, e.g. as favourite01.html, favourite02.html etc.
  2. Create a new html file copying from template.html file you saved earlier. Name it as favourite.html. Create a <div> element with links to each of your favourite things saved in different files.
  3. Now open each of the files containing your favourite things and at the bottom add a link to go back to the menu page (favourite.html).
  4. Visit each link to verify that those are working properly

Task 4 - HTML Validation

As you should learn very quickly in this course, just because a Web page looks correct in your browser on your computer, doesn't mean that it is correct. The easiest way to check whether a Web page is correct is to use a Web validation service.

  1. Go to the W3C MarkUp Validation Service
  2. Using the Browse button, select your Web page, then click on the Validate File (or Validate it!) button.

If all is well, then the validator will announce that your page contains no errors. If it does find any errors, then it will try to tell you why the error occurred and on which line of the file.
