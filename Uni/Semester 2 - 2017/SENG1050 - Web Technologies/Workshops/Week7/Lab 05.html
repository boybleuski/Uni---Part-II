Task 1 - HTML Forms

In this task, you are going to use HTML5 input types and attributes to create a simple form for submitting information to a Web server. Note that we do not need you to know how the CGI script is working. The main aim of this task is for you to become familiar with the various HTML5 form elements, to help you create your form for Assignment 2.

  1. Make a new copy of your template.html file and open it in your favourite plain-text editor (e.g. notepad or textpad).
  2. After adding an appropriate title and level-1 heading, insert form tags into the body of your document: 
    <form>
					</form>
  3. In the opening of the form tag, use attributes method="get" and action="http://www.cs.tut.fi/cgi-bin/run/~jkorpela/echo.cgi".
  4. Within the form, place elements to accept the following information.
    • The applicant's first name (<input type="text">, use placeholder="First name only" and required attributes)
    • The applicant's last name (similar to first name)
    • The applicant's date of birth (<input type="date">, make it required)
    • The applicant's age (<input type="number">, use min, max and value attributes to set the minimum, maximum and default ages)
    • The applicant's email (<input type="email">, use placeholder attribute and make it required)
    • The applicant's website (<input type="url">, use placeholder attribute)
    • The applicant's phone number (<input type="tel">, use placeholder attribute and make it required)
    • The applicant's skill level (<input type="range">, set attributes min="1", max="10" and value="1")
    • The applicant's address (<textarea>, set attributes rows="5" and make it required)
    • The applicant's postcode (<input type="text">, use size="4" and maxlength="4" attributes)
    • The applicant's state ( use <select> and <option> tags. Use one <option> tag for each of the states (NSW,QLD,SA,TAS,VIC,WA) )
    • The applicant's card type (create three <input type="radio"> buttons; use name attribute to ensure that only one of the three card types (visa,mastercard,amex) can be chosen and set visa to be the default choice using the checked attribute)
    • The name on the card (similar to first name/last name)
    • The card number (<input type="number">)
    • The card's security code (<input type="number">, use min and max attributes to enforce the security code to be a 3 digit positive number)
    • The card's expiry date (<input type="month">)
    • For each form control associate a label with <label> tag (use explicit labelling)
    • Group the form controls into four groups with <fieldset> and <legend> tags as shown in the following figure
    There should also be a button to submit the form, and a button to reset the form.

Task 2 - HTML Tables

In this task, you are going to use an HTML table to display some statistical data about the Web.

  1. Make a new copy of your template.html file and open it in your favourite plain-text editor.
  2. After adding an appropriate title and level-1 heading, insert table tags into the body of your document: 
    <table>
					</table>
  3. Inside the table tags, insert <thead>, <tfoot> and <tbody> tags (they must be in that order).
  4. Go to the Internet Systems Consortium site and copy some data from the Distribution of Top-Level Domain Names by Host Count, January 2013. You do not have to copy all the data - just the first 8-10 rows will do.
  5. Use this data to fill the body of your table. Each cell should be enclosed in a table data (<td>) tag, and each row of cells should be enclosed in a table row (<tr>) tag.
  6. Into the head of your table, place a row with the headings of the table. Each cell of this row should use the table heading (<th>) tag.
  7. Into the foot of your table, the totals as listed for the original data.
  8. Finally, add an appropriate <caption> to the table, including a hyperlink to the source of the original data.

Now you can add visual formatting suggestions for your table.

  1. In the content of the <head> tag, place a CSS <style> tag: 
    <style type="text/css">
					</style>
  2. Inside this <style> tag, add CSS rules to suggest visual formatting for your tables.
    • You will have to add class attributes to some tags to differentiate certain rows and columns.
    • You should at least change the text alignment of the numbers to be right aligned, without changing the alignment of the rest of the table's cells (hint: apply text-align: right to the body of the table, then override this with text-align: left on those cells where class="domain" or class="name").
    • You should have different background colours for the cells of the head, the body and the foot of the table.

Don't forget to validate your HTML and CSS.

Task 4 - Image Maps (Client side)

Image maps can be very useful items - you only need 1 image and you can put as many links as you want on it. Using the following free map of Australia we will create an image map for Australian state governments' websites.

Now that we have a free and ready image, we need to turn it into an image map.

  1. Create another copy of template.html and open it in your favourite plain-text editor.
  2. Insert the above image of the map of Australia (you can find it in your 'Lab 05' folder) using the <img> and set usemap attribute to some name e.g. "#aus-govs".
  3. Insert the <map> tag and link it with the inserted image using the name attribute.
  4. Inside the <map> tag insert an <area> tag with shape rect. Choose some coords attribute for rect tag and set href attribute to "https://www.nsw.gov.au/".
  5. Open your html file in browser and check if clicking in the area takes you to the target website.
  6. Now adjust the coordinates so that the area roughly covers the New South Wales region in the map. You might need to use a trial and error approach to choose the right coords.
  7. Repeating Steps 4 to 6 insert another area for Western Australia region with rect shape and link to http://www.wa.gov.au/.
  8. Once again repeat Steps 4 to 6 to insert another area for Tasmania with circle shape and link to http://www.tas.gov.au/.
